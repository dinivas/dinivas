How to contribute to {{ moduleName | uppercase }} module in Dinivas

Following step are required to implement this module integration in Dinivas
You can contribute in one of the requirement project or if you want in the whole module prerequisite.

Ansible role that install {{ moduleName }}

{{ 'https://github.com/dinivas/ansible-role-' + moduleName }}

One of the first step is to write (then submit) an Ansible role that install {{ moduleName | uppercase }} on a given host
Please refer to other existing role if needed.

Packer project to create a pre built image of {{ moduleName | uppercase }}

{{ 'https://github.com/dinivas/packer-' + moduleName }}

The packer project should use the previous Ansible role to build a provider(Openstack, DigitalOcean, AWS, Gcp) image.

Terraform Module to create infrastructure for {{ moduleName }}

{{ 'https://github.com/dinivas/terraform-openstack-' + moduleName }}

A terraform module is then required to help Dinivas to create the infrastructure needed for {{ moduleName | uppercase }}

Dinivas Backend (Api)

https://github.com/dinivas/dinivas

Please make a Pull Request to dinivas in the folder apps/api to handle {{ moduleName | uppercase }} module service, controller, repository

Dinivas Frontend (Console)

https://github.com/dinivas/dinivas

Please make a Pull Request to dinivas in the folder apps/console to setup a Wizard to create {{ moduleName | uppercase }} module. You can refer to existing modules
